# 操作系统智能代理

## 项目简介

操作系统智能代理是一个基于AI的服务器管理工具，允许用户通过自然语言与操作系统进行交互，执行各种系统管理任务。该项目旨在降低系统管理的操作门槛，提供更加直观、友好的人机交互方式。

## 功能特性

### 核心功能

#### 1. 磁盘使用情况监测
- **功能说明**：查询系统各磁盘分区的使用情况
- **支持命令**：
  - Linux：`df -h`
  - Windows：`wmic logicaldisk get caption,size,freespace`
- **输出格式**：显示文件系统、总大小、已用空间、可用空间、使用百分比和挂载点
- **应用场景**：系统维护、磁盘空间管理、存储规划

#### 2. 文件或目录检索
- **功能说明**：根据关键词在系统中搜索文件
- **支持命令**：
  - Linux：`find / -name "*关键词*" -type f`
  - Windows：`dir *关键词* /s /b`
- **搜索参数**：支持文件名匹配、深度限制、文件类型过滤
- **应用场景**：查找配置文件、定位丢失文件、系统审计

#### 3. 进程及端口状态查询
- **功能说明**：查看系统运行进程的详细信息
- **监控指标**：进程ID、名称、CPU使用率、内存使用率
- **排序方式**：按CPU使用率或内存使用率排序
- **应用场景**：性能监控、资源占用分析、故障排查

#### 4. 普通用户的创建与删除
- **功能说明**：管理系统用户账户
- **支持操作**：
  - 创建用户：`useradd 用户名`
  - 删除用户：`userdel -r 用户名`
- **安全措施**：执行前进行风险评估和确认
- **应用场景**：用户管理、权限控制、系统安全

### 安全特性

#### 1. 高风险操作识别
- **识别范围**：用户管理、文件删除、系统配置修改等敏感操作
- **风险评估算法**：基于操作类型、影响范围、权限级别进行综合评估
- **风险等级**：低风险、中风险、高风险
- **识别机制**：关键词匹配、意图分析、操作上下文理解

#### 2. 风险预警
- **预警方式**：Web界面弹窗提示、详细风险说明
- **二次确认**：对高风险操作要求用户明确确认
- **操作限制**：对危险操作进行范围限制，防止误操作
- **紧急停止**：提供操作中断机制，防止意外执行

#### 3. 行为可解释
- **风险说明**：详细解释为什么该操作被判定为高风险
- **影响分析**：说明操作可能带来的系统影响和潜在风险
- **安全建议**：提供替代方案或安全操作建议
- **审计追踪**：记录操作请求和处理过程，便于事后审计

### 交互特性

#### 1. Web界面
- **界面设计**：现代化、响应式布局，适配不同设备
- **交互方式**：文本输入、实时反馈、历史记录
- **视觉效果**：清晰的消息气泡、状态指示、错误提示
- **用户体验**：直观易用，无需技术背景

#### 2. 环境感知
- **自动检测**：系统启动时自动检测操作系统类型和版本
- **环境适配**：根据检测结果选择合适的系统命令
- **信息展示**：在界面顶部显示当前环境信息
- **动态调整**：根据环境变化自动调整功能可用性

#### 3. 跨平台支持
- **支持系统**：Linux（Ubuntu、CentOS、openEuler等）、Windows
- **命令适配**：为不同操作系统提供等效命令
- **统一接口**：用户无需关心底层命令差异
- **兼容性**：支持主流操作系统版本

## 技术架构

### 系统架构

```
+------------------+     +------------------+     +------------------+
|                  |     |                  |     |                  |
|    Web界面       |────>|  自然语言处理模块  |────>|   任务执行器     |
|                  |     |                  |     |                  |
+------------------+     +------------------+     +------------------+
        ^                         ^                         ^
        |                         |                         |
        +-------------------------+                         |
                                  |                         |
                          +------------------+             |
                          |                  |             |
                          |  安全风险控制模块  |─────────────+
                          |                  |
                          +------------------+
                                  ^
                                  |
                          +------------------+
                          |                  |
                          |  环境检测模块    |
                          |                  |
                          +------------------+
```

### 数据流程图

1. **用户输入** → **Web界面**：用户通过浏览器输入自然语言指令
2. **Web界面** → **自然语言处理模块**：将用户输入传递给NLP模块进行意图识别
3. **自然语言处理模块** → **安全风险控制模块**：将识别的意图传递给安全模块进行风险评估
4. **安全风险控制模块** → **Web界面**：如果是高风险操作，返回风险警告给用户
5. **安全风险控制模块** → **任务执行模块**：如果风险可接受，将任务传递给执行模块
6. **任务执行模块** → **环境检测模块**：获取当前操作系统环境信息
7. **环境检测模块** → **任务执行模块**：返回环境信息，用于适配不同操作系统的命令
8. **任务执行模块** → **Web界面**：执行任务并将结果返回给用户

### 模块交互细节

| 模块 | 功能 | 输入 | 输出 |
|------|------|------|------|
| Web界面 | 用户交互 | 自然语言指令 | 执行结果和反馈 |
| 自然语言处理 | 意图识别 | 用户输入文本 | 识别的意图类型 |
| 安全风险控制 | 风险评估 | 意图和用户输入 | 风险级别和警告信息 |
| 任务执行 | 系统操作 | 意图和环境信息 | 执行结果 |
| 环境检测 | 环境信息获取 | 无 | 操作系统信息 |

### 核心模块

1. **自然语言处理模块**：
   - 使用ModelScope API进行意图识别
   - 支持多种系统管理任务的意图分类
   - 处理用户自然语言输入

2. **任务执行模块**：
   - 执行系统管理任务
   - 跨平台命令适配
   - 结果处理和格式化

3. **安全风险控制模块**：
   - 评估操作风险级别
   - 提供风险预警和说明
   - 确保系统操作的安全性

4. **环境检测模块**：
   - 获取操作系统环境信息
   - 检测系统类型和版本
   - 为任务执行提供环境上下文

5. **用户交互模块**：
   - Web界面实现
   - 实时消息处理
   - 结果展示和反馈

## 快速开始

### 环境要求

- Python 3.7或更高版本
- 稳定的网络连接（用于调用ModelScope API）
- 操作系统：Linux或Windows

### 安装步骤

1. **克隆项目**

   ```bash
   git clone <项目地址>
   cd hackathon
   ```

2. **安装依赖**

   ```bash
   pip install -r requirements.txt
   ```

3. **配置API密钥**

   项目已内置ModelScope API配置，无需额外设置。

4. **启动服务**

   ```bash
   python app.py
   ```

   服务将运行在以下地址：
   - http://127.0.0.1:5000
   - http://<本地IP>:5000

### 访问界面

打开浏览器，访问上述地址，即可看到操作系统智能代理的Web界面。

## 使用指南

### 基本操作流程

1. **输入自然语言指令**：在Web界面的输入框中输入您的系统管理需求
2. **智能代理解析意图**：系统自动识别您的意图类型
3. **安全风险评估**：对高风险操作进行评估和预警
4. **执行系统任务**：根据意图执行相应的系统操作
5. **获取执行结果**：查看系统操作的执行结果和详细信息

## 常见问题与故障排除

### 1. API连接失败

**症状**：系统无法识别用户意图
**原因**：可能是ModelScope API连接失败或密钥无效
**解决方案**：
- 检查网络连接是否正常
- 确认API密钥是否正确
- 查看服务器日志了解具体错误信息

### 2. 命令执行失败

**症状**：系统返回执行错误信息
**原因**：可能是权限不足或命令不支持当前操作系统
**解决方案**：
- 确保以管理员权限运行服务
- 检查当前操作系统是否支持该命令
- 查看详细错误信息以确定具体原因

### 3. 界面无响应

**症状**：Web界面无法加载或响应缓慢
**原因**：可能是服务未启动或网络问题
**解决方案**：
- 确认Flask服务是否正常运行
- 检查网络连接
- 尝试刷新浏览器页面

### 4. 意图识别错误

**症状**：系统误解用户意图
**原因**：可能是用户输入表述不够清晰或模型训练不足
**解决方案**：
- 尝试更清晰地表述需求
- 提供更多上下文信息
- 检查模型配置是否正确

## 安全注意事项

1. **API密钥保护**：确保ModelScope API密钥不会被泄露
2. **权限控制**：系统管理操作可能需要管理员权限
3. **风险操作确认**：执行高风险操作前会提示用户确认
4. **日志记录**：建议在生产环境中添加操作日志记录
5. **网络安全**：确保服务运行在安全的网络环境中

## 部署建议

### 开发环境

- 直接运行`python app.py`启动开发服务器
- 开发模式下会自动重载代码变更

### 生产环境

1. **使用WSGI服务器**：
   ```bash
   gunicorn -w 4 -b 0.0.0.0:5000 app:app
   ```

2. **配置反向代理**：
   - 使用Nginx或Apache作为反向代理
   - 配置SSL证书确保通信安全

3. **设置环境变量**：
   - 在生产环境中使用环境变量存储API密钥
   - 避免硬编码敏感信息

4. **监控和日志**：
   - 配置系统监控
   - 设置详细的日志记录
