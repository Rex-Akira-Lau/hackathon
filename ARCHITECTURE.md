# 操作系统智能代理 - 整体架构设计

## 1. 系统概述

### 1.1 项目简介

操作系统智能代理是一款基于大模型的智能系统管理助手，旨在通过自然语言交互实现“去命令行化”的系统管理体验。系统能够精准理解用户意图，自动识别高风险操作，并支撑多步连续任务的编排处理。

### 1.2 核心能力

| 能力维度 | 具体描述 |
|---------|---------|
| 多轮对话 | 支持上下文理解，能够精准利用对话历史信息 |
| 去命令行化 | 通过自然语言交互替代传统命令行操作 |
| 任务编排 | 支持多步连续任务的自动化执行与统一反馈 |
| 安全风控 | 智能识别高风险操作，提供风险预警与二次确认 |
| 跨平台支持 | 支持Ubuntu、CentOS、openEuler等多种Linux发行版及Windows |

## 3. 核心模块设计

### 3.1 NLP处理器 (NLPProcessor)

#### 3.1.1 模块定位

NLP处理器是系统的“理解中枢”，负责将用户的自然语言输入转化为结构化的意图标签，是实现去命令行化体验的关键组件。

#### 3.1.2 核心功能

**意图识别**
- 基于大模型的意图分类
- 支持6种主要意图：磁盘使用、文件搜索、进程状态、用户管理、内存使用、用户指引
- 支持“其他”意图用于通用问答

**上下文理解**
- 接收对话历史作为上下文
- 利用历史信息提升意图识别准确率
- 支持多轮连续对话

**关键词备用识别**
- 当API调用失败时启用
- 基于关键词规则的备用意图识别
- 保证系统在异常情况下的基本可用性

class NLPProcessor:
    def __init__(self, model_config):
        self.client = OpenAI(...)
        self.model_name = model_config['model_name']
    
    def parse_intent(self, user_input: str, history: list) -> str:
        """
        参数:
            user_input: 用户输入的自然语言文本
            history: 对话历史列表，每项包含 role 和 content

        """

### 3.2 任务执行器 (TaskExecutor)

#### 3.2.1 模块定位

任务执行器是系统的“行动中枢”，负责根据识别的意图执行相应的系统命令或操作，是实现系统管理功能的核心组件。

#### 3.2.2 核心功能

**任务分发**
- 根据意图类型分发到对应的处理函数
- 支持磁盘使用、文件搜索、进程状态、用户管理、内存使用、用户指引等任务

**命令执行**
- 跨平台命令执行（Linux/Windows）
- 智能结果格式化输出
- 错误处理与异常捕获

**任务编排**
- 支持多步连续任务的编排处理
- 统一的执行反馈机制
- 基于上下文的连续任务理解

#### 3.2.3 任务分类表

| 意图类型 | 处理函数 | 操作系统支持 |
|---------|---------|-------------|
| 磁盘使用 | get_disk_usage() | Linux/Windows |
| 文件搜索 | search_files() | Linux/Windows |
| 进程状态 | get_process_status() | Linux/Windows |
| 用户管理 | manage_user() | Linux |
| 内存使用 | get_memory_usage() | Linux/Windows |
| 用户指引 | get_user_guide() | Linux/Windows |
| 其他 | handle_other_intent() | Linux/Windows |

### 3.3 安全管理器 (SecurityManager)

#### 3.3.1 模块定位

安全管理器是系统的“安全防线”，负责评估操作风险、生成安全解释、执行审计日志，是保障系统安全运行的关键组件。

#### 3.3.2 核心功能

**风险评估**
- 三级风险分类：high（高风险）、medium（中风险）、low（低风险）
- 高风险操作拦截与确认机制
- 中风险操作记录与监控

**高风险识别维度**
- 系统核心文件/目录删除
- 关键安全配置篡改
- 大范围用户权限变更
- 系统服务停止/禁用
- 系统重启/关机操作

**安全判定解释**
- 为每个安全判定提供清晰的解释
- 操作范围的限制与截断建议
- 拒绝不合理或非法的高风险指令

**安全审计**
- 操作历史记录
- 安全日志持久化
- 操作追溯与审计

#### 3.3.3 风险评估矩阵

| 风险级别 | 判定条件 | 处置方式 |
|---------|---------|---------|
| high | 用户管理意图、包含高风险关键词、涉及核心文件删除 | 拦截并要求二次确认 |
| medium | 查看类操作、启动/重启服务 | 记录日志后执行 |
| low | 查询类操作、信息获取 | 直接执行 |

### 3.4 环境检测器 (EnvironmentDetector)

#### 3.4.1 模块定位

环境检测器是系统的“感知中枢”，负责识别操作系统类型、发行版信息、包管理器和服务管理器，为跨平台支持提供基础数据。

#### 3.4.2 核心功能

**操作系统识别**
- 支持Linux和Windows
- 识别Linux发行版（Ubuntu、CentOS、openEuler等）
- 发行版分类：debian、rhel、suse、arch、other

**包管理器识别**
- apt（Debian系）
- yum/dnf（RHEL系）
- zypper（SUSE系）
- pacman（Arch系）

**服务管理器识别**
- systemd（主流Linux发行版）
- 其他服务管理方式

**网络工具识别**
- ifconfig、ip、ss等网络诊断工具

## 5. API设计

### 5.1 API端点总览

| 端点 | 方法 | 描述 | 请求参数 |
|------|------|------|---------|
| / | GET | 返回主页 | - |
| /api/process | POST | 处理用户请求 | input, history |
| /api/env | GET | 获取环境信息 | - |
| /api/capabilities | GET | 获取系统能力 | - |

### 5.2 /api/process 接口

**请求示例**
```json
POST /api/process
{
    "input": "查看当前磁盘剩余空间",
    "history": [
        {"role": "user", "content": "查看系统进程状态"},
        {"role": "assistant", "content": "进程列表..."}
    ]
}
```

**响应示例 - 成功**
```json
{
    "status": "success",
    "intent": "磁盘使用",
    "result": "磁盘使用情况:\n/dev/sda1: 50GB used / 100GB total",
    "environment": {...}
}
```

**响应示例 - 高风险**
```json
{
    "status": "risk",
    "intent": "用户管理",
    "message": "此操作风险较高，需要确认后执行",
    "details": ["删除用户操作", "涉及系统安全"],
    "security_explanation": {...}
}
```
