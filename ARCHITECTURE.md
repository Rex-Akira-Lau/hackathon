# 操作系统智能代理 - 整体架构设计

## 1. 系统概述

### 1.1 项目简介

操作系统智能代理是一款基于大模型的智能系统管理助手，旨在通过自然语言交互实现"去命令行化"的系统管理体验。系统能够精准理解用户意图，自动识别高风险操作，并支撑多步连续任务的编排处理。

### 1.2 核心能力

| 能力维度 | 具体描述 |
|---------|---------|
| 多轮对话 | 支持上下文理解，能够精准利用对话历史信息 |
| 去命令行化 | 通过自然语言交互替代传统命令行操作 |
| 任务编排 | 支持多步连续任务的自动化执行与统一反馈 |
| 安全风控 | 智能识别高风险操作，提供风险预警与二次确认 |
| 跨平台支持 | 支持Ubuntu、CentOS、Debian、Fedora、Arch、Manjaro等多种Linux发行版及Windows |

## 3. 核心模块设计

### 3.1 NLP处理器 (NLPProcessor)

#### 3.1.1 模块定位

NLP处理器是系统的"理解中枢"，负责将用户的自然语言输入转化为结构化的意图标签，是实现去命令行化体验的关键组件。

#### 3.1.2 核心功能

**意图识别**
- 基于大模型的意图分类
- 支持8种主要意图：磁盘使用、文件搜索、进程状态、用户管理、内存使用、用户指引、网络工具、服务管理
- 支持"其他"意图用于通用问答

**API超时控制**
- 30秒超时设置，避免API调用无限等待
- 异常处理确保服务可用性

**上下文理解**
- 接收对话历史作为上下文
- 利用历史信息提升意图识别准确率
- 支持多轮连续对话

#### 3.1.3 数据结构

```python
class NLPProcessor:
    def __init__(self, model_config):
        self.client = OpenAI(
            base_url=model_config.get('base_url'),
            api_key=model_config.get('api_key'),

    def parse_intent(self, user_input: str, history: list) -> str:
```

### 3.2 任务执行器 (TaskExecutor)

#### 3.2.1 模块定位

任务执行器是系统的"行动中枢"，负责根据识别的意图执行相应的系统命令或操作，是实现系统管理功能的核心组件。

#### 3.2.2 核心功能

**任务分发**
- 根据意图类型分发到对应的处理函数
- 支持磁盘使用、文件搜索、进程状态、用户管理、内存使用、用户指引等任务

**命令执行**
- 跨平台命令执行（Linux/Windows）
- 智能结果格式化输出
- 错误处理与异常捕获
- 统一超时控制（10秒）

**文件搜索超时统一**
- Windows: 10秒超时
- Linux: 10秒超时（从30秒统一）

### 3.3 安全管理器 (SecurityManager)

#### 3.3.1 模块定位

安全管理器是系统的"安全防线"，负责评估操作风险、生成安全解释、执行审计日志，是保障系统安全运行的关键组件。

#### 3.3.2 核心功能

**风险评估**
- 三级风险分类：high（高风险）、medium（中风险）、low（低风险）
- 高风险操作拦截与确认机制

**日志处理器防重复**
- 检查处理器是否已存在，避免重复添加
- 使用 `if not self.logger.handlers:` 保护

**安全审计**
- 操作历史记录
- 安全日志持久化到 `logs/security.log`
- 操作追溯与审计

#### 3.3.3 日志防重复实现

```python
def _init_logger(self):
    self.logger = logging.getLogger('security')
    self.logger.setLevel(logging.INFO)

    # 检查是否已经有处理器，避免重复添加
    if not self.logger.handlers:
        log_dir = 'logs'
        if not os.path.exists(log_dir):
            os.makedirs(log_dir)

        log_file = os.path.join(log_dir, 'security.log')
        file_handler = logging.FileHandler(log_file)
        # ... 设置格式和处理器
        self.logger.addHandler(file_handler)
```

### 3.4 环境检测器 (EnvironmentDetector)

#### 3.4.1 模块定位

环境检测器是系统的"感知中枢"，负责识别操作系统类型、发行版信息、包管理器和服务管理器，为跨平台支持提供基础数据。

#### 3.4.2 核心功能

**操作系统识别**
- 支持Linux和Windows
- 识别Linux发行版信息（从/etc/os-release读取）

**发行版分类检测**
- Debian系：ubuntu, debian, linuxmint, pop, zorin, kali, parrot, deepin, uos
- RHEL系：centos, rhel, fedora, rocky, alma, ol, amazon
- SUSE系：opensuse, sles, suse, suse-linux
- Arch系：arch, manjaro, endeavourouros, garuda, archlabs
- Gentoo系：gentoo, funtoo
- Slackware系：slackware

**包管理器识别**
- apt（Debian系）
- yum/dnf（RHEL系）
- zypper（SUSE系）
- pacman（Arch系）
- emerge（Gentoo系）
- sbopkg/slackpkg（Slackware系）

**服务管理器识别**
- systemd（Debian/RHEL/SUSE/Arch系）
- OpenRC（Gentoo系）
- rc.d（Slackware系）

- 重新初始化NLP和TaskExecutor实例使配置生效

## 6. 安全机制

### 6.1 风险评估

| 风险级别 | 判定条件 | 处置方式 |
|---------|---------|---------|
| high | 用户管理意图、包含高风险关键词、涉及核心文件删除 | 拦截并要求二次确认 |
| medium | 查看类操作、启动/重启服务 | 记录日志后执行 |
| low | 查询类操作、信息获取 | 直接执行 |

### 6.2 高风险识别维度

- 系统核心文件/目录删除
- 关键安全配置篡改
- 大范围用户权限变更
- 系统服务停止/禁用
- 系统重启/关机操作
